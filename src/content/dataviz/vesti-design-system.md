<section class="dataviz-intro">
  <p>
    Vesti Design System began with one question: how can an invisible process
    like AI-assisted thinking become visible, navigable, and trustworthy? I
    treated the project as a single visual system rather than separate outputs,
    extending one language across browser UI, posters, and compact print pieces.
  </p>
  <p>
    The result is a family of artifacts that all communicate the same ideas:
    memory capture, searchable knowledge, networked thinking, and privacy by
    design. The interface prototype handles the product logic, while the print
    materials translate that logic into a public-facing visual identity.
  </p>
</section>

  <div class="dataviz-note-card">

    <h3>Make local-first AI legible</h3>

## Design Language

<section class="dataviz-section-copy">
  <p>
    The interface and print outputs share a deliberately quiet palette and a
    repeated symbolic structure. Large serif headlines establish memory as a
    cultural object rather than a dashboard metric, while smaller sans-serif
    labels handle taxonomy and wayfinding. The pale looping glyph suggests
    accumulation, continuity, and return without becoming a literal diagram.
  </p>
</section>

<div class="dataviz-note-grid">
  <div class="dataviz-note-card">
    <p class="kicker">Typography</p>
    <h3>Large statement, small evidence</h3>
    <p>
      Overscaled wordmarks and headlines create immediate recognition; smaller
      labels carry supporting details such as platform names, features, and QR
      prompts.
    </p>
  </div>
  <div class="dataviz-note-card">
    <p class="kicker">Taxonomy</p>
    <h3>Platforms as chips</h3>
    <p>
      Platform badges turn a multi-model workflow into a readable visual index,
      giving the system a consistent method for tagging origin and context.
    </p>
  </div>
  <div class="dataviz-note-card">
    <p class="kicker">Trust</p>
    <h3>Privacy cues stay visible</h3>
    <p>
      Repeated mentions of local storage, zero-cloud defaults, and QR-driven
      access cues make the product promise legible across digital and print.
    </p>
  </div>
</div>

## Iteration

<section class="dataviz-section-copy">
  <p>
    Earlier versions focused more on brand presence and less on system clarity.
    Later iterations brought the feature taxonomy forward and aligned the poster
    more closely with the product interface, making the print language feel like
    an extension of the UI rather than separate promotion.
  </p>
</section>

## Reflection

<section class="dataviz-section-copy">
  <p>
    What matters most in this project is the continuity between mediums. The UI
    prototype, posters, and bookmark are all doing the same work: turning an
    abstract memory infrastructure into something visible, understandable, and
    shareable. The design system succeeds when the product feels coherent before
    a user reads a single specification.
  </p>
</section>
